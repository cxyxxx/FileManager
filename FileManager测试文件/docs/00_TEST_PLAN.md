# File Manager 测试计划

## 测试目标

验证 File Manager 是否能够支撑真实用户从「混乱文件」到「可理解、可归类、可聚合、可检索」的完整工作流。

## 核心判断

1. 用户能否把一批文件导入工具，并在 Inbox 中看到未整理文件。
2. 用户能否给文件写摘要，通过摘要快速理解文件内容。
3. 用户能否建立 Tag 树，并把文件绑定到一个或多个 Tag。
4. 父 Tag 页面能否默认展示结构模式：直接子 Tag + 直接文件。
5. 父 Tag 页面能否切换聚合模式：全部后代命中文件 + 去重 + matchedTags。
6. 用户能否通过文件名、摘要、Tag 快速检索文件。
7. 重启后文件、摘要、Tag、Saved Query 和聚合结果是否保持一致。

## 测试范围

P0 必测：导入、Inbox、摘要、Tag、结构模式、聚合模式、搜索、重启持久化。

P1 必测：重复文件、同名冲突、版本更新、Tag 移动、Tag 删除、Saved Query 维护。

P2 可选：全文索引、OCR、批量操作、大规模性能、自动摘要、自动推荐 Tag。

## 环境建议

- 每轮测试使用新的 workspace。
- P0 主流程完成后保存一个数据库快照，P1/P2 测试从该快照复制执行。
- 对破坏性操作单独记录开始状态和结束状态。
